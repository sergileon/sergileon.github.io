41  /**
42   * Processes template source files substituting variables and expressions into
43   * placeholders in a template source text to produce the desired output using a
44   * closure based approach. This engine has equivalent functionality to the
45   * {@link groovy.text.SimpleTemplateEngine} but creates the template using
46   * writable closures making it more scalable for large templates.
47   * <p>
48   * Specifically this template engine can handle strings larger than 64k which
49   * still causes problems for the other groovy template engines.
50   * </p>
51   * <p>
52   * The template engine uses JSP style &lt;% %&gt; script and &lt;%= %&gt;
53   * expression syntax or GString style expressions. The variable
54   * '<code>out</code>' is bound to the writer that the template is being written
55   * to.
56   * </p>
57   * Frequently, the template source will be in a file but here is a simple
58   * example providing the template as a string:
59   * <pre>
60   *
61   * def binding = [
62   *   firstname : "Grace",
63   *   lastname  : "Hopper",
64   *   accepted  : true,
65   *   title     : 'Groovy for COBOL programmers'
66   * ]
67   * def text = '''\
68   * Dear <% out.print firstname %> ${lastname},
69   *
70   * We <% if (accepted) out.print 'are pleased' else out.print 'regret' %> \
71   * to inform you that your paper entitled
72   * '$title' was ${ accepted ? 'accepted' : 'rejected' }.
73   *
74   * The conference committee.
75   * '''
76   *
77   * def template = new groovy.text.StreamingTemplateEngine().createTemplate(text)
78   *
79   * print template.make(binding)
80   * </pre>
81   *
82   * This example uses a mix of the JSP style and GString style
83   * placeholders but you can typically use just one style if you wish. Running
84   * this example will produce this output:
85   *
86   * <pre>
87   *
88   * Dear Grace Hopper,
89   *
90   * We are pleased to inform you that your paper entitled
91   * 'Groovy for COBOL programmers' was accepted.
92   *
93   * The conference committee.
94   * </pre>
95   * <br />
96   * <h3>StreamingTemplateEngine as a servlet engine</h3>
97   * The template engine can also be used as the engine for
98   * {@link groovy.servlet.TemplateServlet} by placing the following in your
99   * <code>web.xml</code> file (plus a corresponding servlet-mapping element):
100  * <pre>
101  *
102  * &lt;servlet&gt;
103  *   &lt;servlet-name&gt;StreamingTemplate&lt;/servlet-name&gt;
104  *   &lt;servlet-class&gt;groovy.servlet.TemplateServlet&lt;/servlet-class&gt;
105  *   &lt;init-param&gt;
106  *     &lt;param-name&gt;template.engine&lt;/param-name&gt;
107  *     &lt;param-value&gt;groovy.text.StreamingTemplateEngine&lt;/param-value&gt;
108  *   &lt;/init-param&gt;
109  * &lt;/servlet&gt;
110  * </pre> In this case, your template source file should be HTML with the
111  * appropriate embedded placeholders.
112  *
113  * <h3>Debugging Template Code</h3>
114  * <p>The template engine makes an effort to throw descriptive exceptions with
115  * context lines, ie:
116  * <pre>
117  *  groovy.text.TemplateExecutionException: Template parse error at line 4:
118  *           3: We <% if (accepted) out.print 'are pleased' else out.print 'regret' %> to inform you that your paper entitled
119  *       --> 4: '$txitle' was ${ accepted ? 'accepted' : 'rejected' }.
120  *           5:
121  * at test.run(test.groovy:18)
122  *
123  * Caused by: groovy.lang.MissingPropertyException: No such property: txitle for class: groovy.tmp.templates.StreamingTemplateScript1
124  * ... 1 more
125  * </pre>
126  * and sanitize the exceptions to make things readable.
127  * </p>
128  * <p>When the exceptions are not enough, it might sometimes be useful to view the actual script source
129  * generated by the template engine. This would conceptually be equivalent to viewing the
130  * .java file generated for a jsp page. The source is not currently very readable and
131  * until we get a built in groovy code pretty printer, we will probably continue to opt for compactness
132  * rather than readability.</p>
133  *
134  * <p>With that being said, viewing the source might still have some value. For this reason the script
135  * source is accessible via the template.scriptSource property, i.e.:
136  * <pre>
137  *     println template.scriptSource
138  * </pre>
139  * In the above example.
140  * </p>
141  *
142  * @author mbjarland@gmail.com
143  * @author Matias Bjarland
144  */

169     /**
170      * <p>Creates a template instance using the template source from the provided Reader.</p>
171      *
172      * <p>The template can be applied repeatedly on different bindings to produce custom
173      * output.</p>
174      *
175      *
176      * <strong>Technical detail</strong><br />
177      * Under the hood the returned template is represented as a four argument
178      * closure where the three first arguments are {@link groovy.lang.Closure#curry curried} in
179      * while generating the template. <br />
180      * <br />
181      * In essence we start with a closure on the form:
182      *
183      * <pre>
184      *   { parentClass, stringSectionList, binding, out ->
185      *      //code generated by parsing the template data
186      *   }                                                                             *
187      * </pre>
188      *
189      * , we then curry in the parentClass and stringSectionList arguments so that the StreamingTemplate
190      * instance returned from 'createTemplate' internally contains a template closure on the form:
191      *
192      * <pre>
193      *   { binding, out ->
194      *      //code generated by parsing the template data
195      *   }                                                                             *
196      * </pre>
197      *
198      * Calling template.make(binding), curries in the 'binding' argument:
199      *
200      * <pre>
201      *   public Writable make(final Map map) {
202      *     final Closure template = this.template.curry(new Object[]{map});
203      *     return (Writable) template;
204      *   }
205      * </pre>
206      *
207      * which only leaves the 'out' argument unbound. The only method on the {@link groovy.lang.Writable writable} interface is
208      * {@link groovy.lang.Writable#writeTo writeTo(Writer out)} so groovy rules about casting a closure to a one-method-interface
209      * apply and the above works. I.e. we return the now one argument closure as the Writable
210      * which can be serialized to System.out, a file, etc according to the Writable interface contract.
211      * </p>
212      * @see groovy.text.TemplateEngine#createTemplate(java.io.Reader)
213      */
214     @Override
215     public Template createTemplate(final Reader reader) throws CompilationFailedException, ClassNotFoundException, IOException {
216         return new StreamingTemplate(reader, parentLoader);
217     }

227         private static final String SCRIPT_HEAD
228                 = "package groovy.tmp.templates;"
229                 + "def getTemplate() { "
230                     //the below params are:
231                     //  _p - parent class, for handling exceptions
232                     //  _s - sections, string sections list
233                     //  _b - binding map
234                     //  out - out stream
235                     //the three first parameters will be curried in as we move along
236                 +   "return { _p, _s, _b, out -> "
237                 +     "int _i = 0;"
238                 +     "try {"
239                 +       "delegate = new Binding(_b);";

243          */

245                 =     "} catch (Throwable e) { "
246                 +       "_p.error(_i, _s, e);"
247                 +     "}"
248                 +   "}.asWritable()"
249                 + "}";

299         /**
300          * A StringSection represent a section in the template source
301          * with only string data (i.e. no branching, GString references, etc).
302          * As an example, the following template string:
303          *
304          * <pre>
305          * Alice why is a $bird like a writing desk
306          * </pre>
307          *
308          * Would produce a string section "Alice why is a " followed by
309          * a dollar identifier expression followed by another string
310          * section " like a writing desk".
311          */
312         private static final class StringSection {
313             StringBuilder data;
314             Position firstSourcePosition;
315             Position lastSourcePosition;
316             Position lastTargetPosition;
317 
318             private StringSection(Position firstSourcePosition) {
319                 this.data = new StringBuilder();
320                 this.firstSourcePosition = new Position(firstSourcePosition);
321             }
322 
323             @Override
324             public String toString() {

329         /**
330          * Called to handle the ending of a string section.
331          *
332          * @param sections The list of string sections. The current section gets added to this section.
333          * @param currentSection The current string section.
334          * @param templateExpressions Template expressions
335          * @param lastSourcePosition The last read position in the source template stream.
336          * @param targetPosition The last written to position in the target script stream.
337          */
338         private void finishStringSection(List<StringSection> sections, StringSection currentSection,
339                                          StringBuilder templateExpressions,
340                                          Position lastSourcePosition, Position targetPosition) {
341             //when we get exceptions from the parseXXX methods in the main loop, we might try to
342             //re-finish a section
343             if (currentSection.lastSourcePosition != null) {
344                 return;
345             }
346             currentSection.lastSourcePosition = new Position(lastSourcePosition);
347             sections.add(currentSection);
348             append(templateExpressions, targetPosition, "out<<_s[_i=" + index++ + "];");
349             currentSection.lastTargetPosition = new Position(targetPosition.row, targetPosition.column);
350         }

352         public void error(int index, List<StringSection> sections, Throwable e) throws Throwable {
353             int i = Math.max(0, index);
354             StringSection precedingSection = sections.get(i);
355             int traceLine = -1;
356             for (StackTraceElement element : e.getStackTrace()) {
357                 if (element.getClassName().contains(TEMPLATE_SCRIPT_PREFIX)) {
358                     traceLine = element.getLineNumber();
359                     break;
360                 }
361             }
362 
363             if (traceLine != -1) {
364                 int actualLine = precedingSection.lastSourcePosition.row + traceLine - 1;
365                 String message = "Template execution error at line " + actualLine + ":\n" + getErrorContext(actualLine);
366                 TemplateExecutionException unsanitized = new TemplateExecutionException(actualLine, message, StackTraceUtils.sanitize(e));
367                 throw StackTraceUtils.sanitize(unsanitized);
368             } else {
369                 throw e;
370             }
371         }

373         private int getLinesInSource() throws IOException {
374             int result = 0;
375 
376             LineNumberReader reader = null;
377             try {
378                 reader = new LineNumberReader(new StringReader(templateSource.toString()));
379                 reader.skip(Long.MAX_VALUE);
380                 result = reader.getLineNumber();
381             } finally {
382                 if (reader != null) {
383                     reader.close();
384                 }
385             }
386 
387             return result;
388         }

390         private String getErrorContext(int actualLine) throws IOException {
391             int minLine = Math.max(0, actualLine -1);
392             int maxLine = Math.min(getLinesInSource(), actualLine + 1);
393 
394             LineNumberReader r = new LineNumberReader(new StringReader(templateSource.toString()));
395             int lineNr;
396             StringBuilder result = new StringBuilder();
397             while ((lineNr = r.getLineNumber()+1) <= maxLine) {
398                 String line = r.readLine();
399                 if (lineNr < minLine) continue;
400 
401                 String nr = Integer.toString(lineNr);
402                 if (lineNr == actualLine) {
403                     nr = " --> " + nr;
404                 }
405 
406                 result.append(padLeft(nr, 10));
407                 result.append(": ");
408                 result.append(line);
409                 result.append('\n');
410             }
411 
412             return result.toString();
413         }

415         private String padLeft(String s, int len) {
416             StringBuilder b = new StringBuilder(s);
417             while (b.length() < len) b.insert(0, " ");
418             return b.toString();
419         }

421         /**
422          * Turn the template into a writable Closure. When executed the closure
423          * evaluates all the code embedded in the template and then writes a
424          * GString containing the fixed and variable items to the writer passed
425          * as a parameter
426          * <p/>
427          * For example:
428          * <pre>
429          * '<%= "test" %> of expr and <% test = 1 %>${test} script.'
430          * </pre>
431          * would compile into:
432          * <pre>
433          * { out -> out << "${"test"} of expr and "; test = 1 ; out << "${test} script."}.asWritable()
434          * </pre>
435          * @param source A reader into the template source data
436          * @param parentLoader A class loader we use
437          * @throws CompilationFailedException
438          * @throws ClassNotFoundException
439          * @throws IOException
440          */
441         StreamingTemplate(final Reader source, final ClassLoader parentLoader) throws CompilationFailedException, ClassNotFoundException, IOException {
442             final StringBuilder target = new StringBuilder();
443             List<StringSection> sections = new ArrayList<StringSection>();
444             Position sourcePosition = new Position(1, 1);
445             Position targetPosition = new Position(1, 1);
446             Position lastSourcePosition = new Position(1, 1);
447             StringSection currentSection = new StringSection(sourcePosition);
448             templateSource = new StringBuilder();
449 
450             //we use the lookAhead to make sure that a template file ending in say "abcdef\\"
451             //will give a result of "abcdef\\" even though we have special handling for \\
452             StringBuilder lookAhead = new StringBuilder(10);
453 
454             append(target, targetPosition, SCRIPT_HEAD);
455             try {
456                 int skipRead = -1;
457                 //noinspection InfiniteLoopStatement
458                 while (true) {
459                     lastSourcePosition.set(sourcePosition);
460 
461                     int c = (skipRead != -1) ? skipRead : read(source, sourcePosition, lookAhead);
462                     skipRead = -1;
463 
464                     if (c == '\\') {
465                         handleEscaping(source, sourcePosition, currentSection, lookAhead);
466                         continue;
467                     } else if (c == '<') {
468                         c = read(source, sourcePosition, lookAhead);
469                         if (c == '%') {
470                             c = read(source, sourcePosition);
471                             clear(lookAhead);
472                             if (c == '=') {
473                                 finishStringSection(sections, currentSection, target, lastSourcePosition, targetPosition);
474                                 parseExpression(source, target, sourcePosition, targetPosition);
475                                 currentSection = new StringSection(sourcePosition);
476                                 continue;
477                             } else {
478                                 finishStringSection(sections, currentSection, target, lastSourcePosition, targetPosition);
479                                 parseSection(c, source, target, sourcePosition, targetPosition);
480                                 currentSection = new StringSection(sourcePosition);
481                                 continue;
482                             }
483                         } else {
484                             currentSection.data.append('<');
485                         }
486                     } else if (c == '$') {
487                         c = read(source, sourcePosition);
488                         clear(lookAhead);
489                         if (c == '{') {
490                             finishStringSection(sections, currentSection, target, lastSourcePosition, targetPosition);
491                             parseDollarCurlyIdentifier(source, target, sourcePosition, targetPosition);
492                             currentSection = new StringSection(sourcePosition);
493                             continue;
494                         } else if (Character.isJavaIdentifierStart(c)) {
495                             finishStringSection(sections, currentSection, target, lastSourcePosition, targetPosition);
496                             skipRead = parseDollarIdentifier(c, source, target, sourcePosition, targetPosition);
497                             currentSection = new StringSection(sourcePosition);
498                             continue;
499                         } else {
500                             currentSection.data.append('$');
501                         }
502                     }
503                     currentSection.data.append((char) c);
504                     clear(lookAhead);
505                 }
506             } catch (FinishedReadingException e) {
507                 if (lookAhead.length() > 0) {
508                     currentSection.data.append(lookAhead);
509                 }
510                 //Ignored here, just used for exiting the read loop. Yeah I know we don't like
511                 //empty catch blocks or expected behavior trowing exceptions, but this just cleaned out the code
512                 //_so_ much that I thought it worth it...this once -Matias Bjarland 20100126
513             }
514 
515             finishStringSection(sections, currentSection, target, sourcePosition, targetPosition);
516             append(target, targetPosition, SCRIPT_TAIL);
517 
518             scriptSource = target.toString();
519 
520             this.template = createTemplateClosure(sections, parentLoader, target);
521         }
522 
523         private static void clear(StringBuilder lookAhead) {
524             lookAhead.delete(0, lookAhead.length());
525         }

527         private void handleEscaping(final Reader source,
528                                     final Position sourcePosition,
529                                     final StringSection currentSection,
530                                     final StringBuilder lookAhead) throws IOException, FinishedReadingException {
531             //if we get here, we just read in a back-slash from the source, now figure out what to do with it
532             int c = read(source, sourcePosition, lookAhead);
533 
534             /*
535              The _only_ special escaping this template engine allows is to escape the sequences:
536              ${ and <% and potential slashes in front of these. Escaping in any other sections of the
537              source string is ignored. The following is a source -> result mapping of a few values, assume a
538              binding of [alice: 'rabbit'].
539 
540              Note: we don't do java escaping of slashes in the below
541              example, i.e. the source string is what you would see in a text editor when looking at your template
542              file: 
543              source string     result
544              'bob'            -> 'bob'
545              '\bob'           -> '\bob'
546              '\\bob'          -> '\\bob'
547              '${alice}'       -> 'rabbit'
548              '\${alice}'      -> '${alice}'
549              '\\${alice}'     -> '\rabbit'
550              '\\$bob'         -> '\\$bob'
551              '\\'             -> '\\'
552              '\\\'             -> '\\\'
553              '%<= alice %>'   -> 'rabbit'
554              '\%<= alice %>'  -> '%<= alice %>'
555              */
556             if (c == '\\') {
557                 //this means we have received a double backslash sequence
558                 //if this is followed by ${ or <% we output one backslash
559                 //and interpret the following sequences with groovy, if followed by anything
560                 //else we output the two backslashes and continue as usual 
561                 source.mark(3);
562                 int d = read(source, sourcePosition, lookAhead);
563                 c = read(source, sourcePosition, lookAhead);
564                 clear(lookAhead);
565                 if ((d == '$' && c == '{') ||
566                     (d == '<' && c == '%')) {
567                     source.reset();
568                     currentSection.data.append('\\');
569                     return;
570                 } else {
571                     currentSection.data.append('\\');
572                     currentSection.data.append('\\');
573                     currentSection.data.append((char) d);
574                 }
575             } else if (c == '$') {
576                 c = read(source, sourcePosition, lookAhead);
577                 if (c == '{') {
578                     currentSection.data.append('$');
579                 } else {
580                     currentSection.data.append('\\');
581                     currentSection.data.append('$');
582                 }
583             } else if (c == '<') {
584                 c = read(source, sourcePosition, lookAhead);
585                 if (c == '%') {
586                     currentSection.data.append('<');
587                 } else {
588                     currentSection.data.append('\\');
589                     currentSection.data.append('<');
590                 }
591             } else {
592                 currentSection.data.append('\\');
593             }
594 
595             currentSection.data.append((char) c);
596             clear(lookAhead);
597         }

599         private Closure createTemplateClosure(List<StringSection> sections, final ClassLoader parentLoader, StringBuilder target) throws ClassNotFoundException {
600             final GroovyClassLoader loader = AccessController.doPrivileged(new PrivilegedAction<GroovyClassLoader>() {
601                 public GroovyClassLoader run() {
602                     return new GroovyClassLoader(parentLoader);
603                 }
604             });
605             final Class groovyClass;
606             try {
607                 groovyClass = loader.parseClass(new GroovyCodeSource(target.toString(), TEMPLATE_SCRIPT_PREFIX + counter++ + ".groovy", "x"));
608             } catch (MultipleCompilationErrorsException e) {
609                 throw mangleMultipleCompilationErrorsException(e, sections);
610 
611             } catch (Exception e) {
612                 throw new GroovyRuntimeException("Failed to parse template script (your template may contain an error or be trying to use expressions not currently supported): " + e.getMessage());
613             }
614 
615             Closure result;
616             try {
617                 final GroovyObject object = (GroovyObject) groovyClass.newInstance();
618                 Closure chicken = (Closure) object.invokeMethod("getTemplate", null);
619                 //bind the two first parameters of the generated closure to this class and the sections list
620                 result = chicken.curry(this, sections);
621             } catch (InstantiationException e) {
622                 throw new ClassNotFoundException(e.getMessage());
623             } catch (IllegalAccessException e) {
624                 throw new ClassNotFoundException(e.getMessage());
625             }
626 
627             return result;
628         }

630         /**
631          * Parses a non curly dollar preceded identifier of the type
632          * '$bird' in the following template example:
633          *
634          * <pre>
635          * Alice why is a $bird like a writing desk
636          * </pre>
637          *
638          * which would produce the following template data:
639          *
640          * <pre>
641          * out << "Alice why is a ";
642          * out << bird;
643          * out << " like a writing desk";
644          * </pre>
645          *
646          * This method is given the 'b' in 'bird' in argument c, checks if it is a valid
647          * java identifier start (we assume groovy did not mangle the java
648          * identifier rules). If so it proceeds to parse characters from the input
649          * until it encounters a non-java-identifier character. At that point
650          *
651          * @param c The first letter of the potential identifier, 'b' in the above example
652          * @param reader The reader reading from the template source
653          * @param target The target groovy script source we write to
654          * @param sourcePosition The reader position in the source stream
655          * @param targetPosition The writer position in the target stream
656          * @return true if a valid dollar preceded identifier was found, false otherwise. More
657          *         specifically, returns true if the first character after the dollar sign is
658          *         a valid java identifier. Note that the dollar curly syntax is handled by
659          *         another method.
660          *
661          * @throws IOException
662          * @throws FinishedReadingException If we encountered the end of the source stream.
663          */
664         private int parseDollarIdentifier(int c ,
665                                           final Reader reader,
666                                           final StringBuilder target,
667                                           final Position sourcePosition,
668                                           final Position targetPosition) throws IOException, FinishedReadingException {
669             append(target, targetPosition, "out<<");
670             append(target, targetPosition, (char) c);
671 
672             while (true) {
673                 c = read(reader, sourcePosition);
674                 if (!Character.isJavaIdentifierPart(c) || c == '$') {
675                     break;
676                 }
677                 append(target, targetPosition, (char) c);
678             }
679 
680             append(target, targetPosition, ";");
681 
682             return c;
683         }

685         /**
686          * Parses a dollar curly preceded identifier of the type
687          * '${bird}' in the following template example:
688          *
689          * <pre>
690          * Alice why is a ${bird} like a writing desk
691          * </pre>
692          *
693          * which would produce the following template data:
694          *
695          * <pre>
696          * out << "Alice why is a ";
697          * out << """${bird}""";
698          * out << " like a writing desk";
699          * </pre>
700          *
701          * This method is given the 'b' in 'bird' in argument c, checks if it is a valid
702          * java identifier start (we assume groovy did not mangle the java
703          * identifier rules). If so it proceeds to parse characters from the input
704          * until it encounters a non-java-identifier character. At that point
705          *
706          * @param reader The reader reading from the template source
707          * @param target The target groovy script source we write to
708          * @param sourcePosition The reader position in the source stream
709          * @param targetPosition The writer position in the target stream
710          * @throws IOException
711          * @throws FinishedReadingException
712          */
713         private void parseDollarCurlyIdentifier(final Reader reader,
714                                                 final StringBuilder target,
715                                                 final Position sourcePosition,
716                                                 final Position targetPosition) throws IOException, FinishedReadingException {
717             append(target, targetPosition, "out<<\"\"\"${");
718 
719             while (true) {
720                 int c = read(reader, sourcePosition);
721                 append(target, targetPosition, (char) c);
722                 if (c == '}') break;
723             }
724 
725             append(target, targetPosition, "\"\"\";");
726         }

728         /**
729          * Parse a &lt;% .... %&gt; section if we are writing a GString close and
730          * append ';' then write the section as a statement
731          */
732         private void parseSection(final int pendingC,
733                 final Reader reader,
734                 final StringBuilder target,
735                 final Position sourcePosition,
736                 final Position targetPosition) throws IOException, FinishedReadingException {
737             //the below is a quirk, we do this so that every non-string-section is prefixed by
738             //the same number of characters (the others have "out<<\"\"\"${"), this allows us to
739             //figure out the exception row and column later on
740             append(target, targetPosition, "          ");
741             append(target, targetPosition, (char) pendingC);
742 
743             while (true) {
744                 int c = read(reader, sourcePosition);
745                 if (c == '%') {
746                     c = read(reader, sourcePosition);
747                     if (c == '>') break;
748                     append(target, targetPosition, '%');
749                 }
750                 append(target, targetPosition, (char) c);
751             }
752 
753             append(target, targetPosition, ';');
754         }

756         /**
757          * Parse a &lt;%= .... %&gt; expression
758          */
759         private void parseExpression(final Reader reader,
760                 final StringBuilder target,
761                 final Position sourcePosition,
762                 final Position targetPosition) throws IOException, FinishedReadingException {
763             append(target, targetPosition, "out<<\"\"\"${");
764 
765             while (true) {
766                 int c = read(reader, sourcePosition);
767                 if (c == '%') {
768                     c = read(reader, sourcePosition);
769                     if (c == '>') break;
770                     append(target, targetPosition, '%');
771                 }
772                 append(target, targetPosition, (char) c);
773             }
774 
775             append(target, targetPosition, "}\"\"\";");
776         }

790         /*
791          * Create groovy assertion style error message for template error. Example:
792          *
793          * Error parsing expression on line 71 column 15, message: no such property jboss for for class DUMMY
794          * templatedata${jboss}templateddatatemplateddata
795          *             ^------^
796          *                 |
797          *           syntax error
798          */
799         private RuntimeException mangleMultipleCompilationErrorsException(MultipleCompilationErrorsException e, List<StringSection> sections) {
800             RuntimeException result = e;
801 
802             ErrorCollector collector = e.getErrorCollector();
803             @SuppressWarnings({"unchecked"})
804             List<Message> errors = (List<Message>) collector.getErrors();
805             if (errors.size() > 0) {
806                 Message firstMessage = errors.get(0);
807                 if (firstMessage instanceof SyntaxErrorMessage) {
808                     @SuppressWarnings({"ThrowableResultOfMethodCallIgnored"})
809                     SyntaxException syntaxException = ((SyntaxErrorMessage) firstMessage).getCause();
810                     Position errorPosition = new Position(syntaxException.getLine(), syntaxException.getStartColumn());
811 
812                     //find the string section which precedes the row/col of the thrown exception
813                     StringSection precedingSection = findPrecedingSection(errorPosition, sections);
814 
815                     //and now use the string section to mangle the line numbers so that they refer to the
816                     //appropriate line in the source template data
817                     if (precedingSection != null) {
818                         //if the error was thrown on the same row as where the last string section
819                         //ended, fix column value
820                         offsetPositionFromSection(errorPosition, precedingSection);
821                         //the below being true indicates that we had an unterminated ${ or <% sequence and
822                         //the column is thus meaningless, we reset it to where the %{ or <% starts to at
823                         //least give the user a sporting chance
824                         if (sections.get(sections.size() - 1) == precedingSection) {
825                             errorPosition.column = precedingSection.lastSourcePosition.column;
826                         }
827 
828                         String message = mangleExceptionMessage(e.getMessage(), errorPosition);
829                         result = new TemplateParseException(message, e, errorPosition.row, errorPosition.column);
830                     }
831                 }
832             }
833 
834             return result;
835         }

837         private String mangleExceptionMessage(String original, Position p) {
838             String result = original;
839             int index = result.indexOf("@ line ");
840             if (index != -1) {
841                 result = result.substring(0, index);
842             }
843 
844             int count = 0;
845             index = 0;
846             for (char c : result.toCharArray()) {
847                 if (c == ':') {
848                     count++;
849                     if (count == 3) {
850                         result = result.substring(index + 2);
851                         break;
852                     }
853                 }
854                 index++;
855             }
856 
857             String msg = "Template parse error '" + result + "' at line " + p.row + ", column " + p.column;
858             try {
859                 msg += "\n" + getErrorContext(p.row);
860             } catch (IOException e) {
861                 //we opt for not doing anthing here...we just do not get context if
862                 //this happens
863             }
864 
865             return msg;
866         }

868         private void offsetPositionFromSection(Position p, StringSection s) {
869             if (p.row == s.lastTargetPosition.row) {
870                 //The number 8 below represents the number of characters in the header of a non-string-section such as
871                 //<% ... %>. A section like this is represented in the target script as:
872                 //out<<"""......."""
873                 //12345678
874                 p.column -= s.lastTargetPosition.column + 8;
875                 p.column += s.lastSourcePosition.column;
876             }
877 
878             p.row += s.lastSourcePosition.row - 1;
879         }

881         private StringSection findPrecedingSection(Position p, List<StringSection> sections) {
882             StringSection result = null;
883             for (StringSection s : sections) {
884                 if (s.lastTargetPosition.row > p.row
885                         || (s.lastTargetPosition.row == p.row && s.lastTargetPosition.column > p.column)) {
886                     break;
887                 }
888                 result = s;
889             }
890 
891             return result;
892         }

918         // SEE BELOW
919         boolean useLastRead = false;
920         private int lastRead = -1;
921 
922         /* All \r\n sequences are treated as a single \n. By doing this we
923          * produce the same output as the GStringTemplateEngine. Otherwise, some
924          * of our output is on a newline when it should not be.
925          *
926          * Instead of using a pushback reader, we just keep a private instance
927          * variable 'lastRead'.
928          */
929         private int read(final Reader reader, Position position) throws IOException, FinishedReadingException {
930             int c;
931 
932             if (useLastRead) {
933                 // use last one if we stored a character
934                 c = lastRead;
935                 // reset last
936                 useLastRead = false;
937                 lastRead = -1;
938             } else {
939                 c = read(reader);
940                 if (c == '\r') {
941                     // IF CRLF JUST KEEP LF
942                     c = read(reader);
943                     if (c != '\n') {
944                         // ELSE keep original char
945                         // and pushback the one we just read
946                         lastRead = c;
947                         useLastRead = true;
948                         c = '\r';
949                     }
950                 }
951             }
952 
953             if (c == -1) {
954                 throw finishedReadingException;
955             }
956 
957             if (c == '\n') {
958                 position.row++;
959                 position.column = 1;
960             } else {
961                 position.column++;
962             }
963 
964             return c;
965         }

967         private int read(final Reader reader) throws IOException {
968             int c = reader.read();
969             templateSource.append((char) c);
970             return c;
971         }
